# BookVerse: локальный умный поиск по книгам

Решение для кейса **«Умный поиск по книгам»** в полностью офлайн-режиме.

## Что реализовано

- загрузка книг `.txt` и `.fb2` прямо в браузере;
- хранение книг локально (LocalStorage), без backend и без внешних API;
- поиск фрагментов по книге или по всем книгам;
- ответы на вопросы по тексту с цитатами-основаниями;
- честные сообщения, если главы/ответа нет.

## Архитектура

- **Frontend-only**: статический сайт (`index.html`, `library.html`, `reader.html`);
- **Без API**: нет зависимости от `http://localhost:8000` и любых внешних LLM;
- **Алгоритмы в браузере**: токенизация, чанкинг текста, ранжирование, extractive-ответ.

Ключевой модуль: `scripts/localBooks.js`.

## Гайд на запуск

Нужен только статический сервер.

1. Открой терминал:

```bash
cd "/Users/kirill/Documents/Code/BookVerse (3)"
python3 -m http.server 5500
```

2. Открой в браузере:

- `http://127.0.0.1:5500/index.html`

## Как проверить функциональность

1. Нажми **«Добавить книгу»** и загрузи `.txt` или `.fb2`.
2. Открой книгу кнопкой **«Открыть/Читать»**.
3. В меню открой **«Инструменты»**.
4. Проверь:
- поиск: `найди где говорится о ...`;
- вопросы: `что происходит в начале`, `что во 2-й главе`, `какая основная тема`.

## Важные заметки

- Книги и прогресс хранятся в браузере текущего устройства.
- При очистке LocalStorage данные библиотеки удалятся.
- Проект intentionally не использует backend/API/LLM.
